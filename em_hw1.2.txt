APT는 데비안 GNU/리눅스에서 소프트웨어 설치 또는 제거 작업을 할 때 사용하는 소프트웨어

사용법 : apt-get [옵션] 명령
		  -> 주로 사용하는 명령은 update 와 install

	ex.) sudo apt-get update -> 패키지 소스 리스트 갱신
	     sudo apt-get upgrade -> 설치된 패키지 업그레이드
		 sudo apt-get install (패키지이름) -> 패키지 설치
		 sudo apt-get remove (패키지이름) -> 패키지 제거
		 sudo apt-get reinstall (패키지이름) -> 패키지 재설치 (소프트웨어 오류)
		  
-------명령-------
   update - 새 패키지 목록 가져오기
   upgrade - 업그레이드 실행
   install - 새 피키지 설치 (패키지 이름은 확장자를 제외함)
   remove - 패키지 제거
   autoremove - 사용하지 않는 모든 패키지를 자동으로 제거
   purge - 패키지와 설정 파일을 함께 제거
   source - 소스 압축 파일 다운로드
   build-dep - 소스 패키지의 빌드 의존성 설정
   dist-upgrade - 배포판 업그레이드
   dselect-upgrade - dselect 선택 따르기
   clean - 다운로드한 압축 파일 지우기
   autoclean - 다운로드한 압축 파일 중 오래된 것 지우기
   check -의존성이 깨진 패키지를 확인
   changelog - 주어진 패키지의 바뀐 내용 목록을 다운로드한 후 표시
   download - 현재 디렉터리로 바이너리 패키지 다운로드
------------------

-------옵션-------

  -h  도움말
  -q  기록할 수 있는 출력
  -qq 이외의 메시지 표시하지 않기
  -d  압축 파일을 설치하거나 압축 해제하지 않고 다운로드만
  -s  동작 없음. 명령 시뮬레이션 실행
  -y  모든 질문을 표시하지 않고 예라고 대답
  -f  망가진 의존성 패키지가 있는 시스템을 즉시 정정
  -m  압축 파일을 찾을 수 없어도 계속 진행
  -u  업그레이드한 패키지의 목록도 표시
  -b  소스 패키지를 가져온 후 빌드
  -V  자세한 버전 번호 표시
